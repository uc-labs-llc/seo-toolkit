File Structure & Architecture

/analytics/
user.html - Demo user page
dashboard.html - Analytics dashboard
tracker.js - Client-side tracking
log_endpoint.php - Data processing API
get_log_data.php - Data retrieval API
analytics.log - Data storage (auto-generated)
.htaccess - Security configuration

Core Files Explained

tracker.js

Client-side JavaScript that tracks user interactions, manages sessions, and sends data to the server.

log_endpoint.php

PHP API endpoint that receives, validates, and stores analytics data in the log file.

dashboard.html

Interactive dashboard that visualizes analytics data with charts, metrics, and real-time updates.

Data Flow

  1. 1 User visits website with tracker.js
  2. 2 Tracker collects data and sends to log_endpoint.php
  3. 3 PHP validates and stores data in analytics.log
  4. 4 Dashboard reads data via get_log_data.php
  5. 5 Data is visualized in real-time charts and metrics

Apache Web Server Setup Guide

Prerequisites: Ubuntu Server with Apache2 installed. Basic terminal knowledge required.

1 Enable Required Apache Modules

Enable mod_rewrite for .htaccess support and clean URLs.

# Enable rewrite module
sudo a2enmod rewrite

# Enable headers module for security
sudo a2enmod headers

2 Configure Virtual Host

Edit your site configuration to enable .htaccess and set proper permissions.

# Edit your site configuration
sudo nano /etc/apache2/sites-available/000-default.conf

Add this inside your VirtualHost block:

<!-- Enable .htaccess in analytics directory -->
<Directory /var/www/html/analytics>
    AllowOverride All
    Require all granted
</Directory>

3 Set File Permissions

Ensure proper ownership and permissions for security.

# Navigate to your directory
cd /var/www/html/analytics

# Set Apache as owner
sudo chown -R www-data:www-data .

# Set secure permissions
sudo chmod 644 *.html *.js *.php
sudo chmod 640 analytics.log

4 Test and Restart Apache

Validate configuration and apply changes.

# Test configuration syntax
sudo apache2ctl configtest

# If syntax is OK, restart Apache
sudo systemctl restart apache2

# Enable Apache to start on boot
sudo systemctl enable apache2

5 Verify Installation

Test that everything is working correctly.

# Test dashboard access
curl -I http://localhost/analytics/dashboard.html

# Test API endpoint
curl -X POST http://localhost/analytics/log_endpoint.php \
  -H "Content-Type: application/json" \
  -d '{"action":"TEST"}'

.htaccess Security Configuration

Create this file in your /analytics/ directory to protect sensitive files:

# Block direct access to analytics log file
<Files "analytics.log">
    Order Allow,Deny
    Deny from all
</Files>

# Protect PHP endpoints
<Files "log_endpoint.php">
    Order Allow,Deny
    Allow from all
    # Add rate limiting here if needed
</Files>

# Password protect dashboard (optional)
<Files "dashboard.html">
    AuthType Basic
    AuthName "Analytics Dashboard"
    AuthUserFile /etc/apache2/.htpasswd
    Require valid-user
</Files>

Usage Guide & Implementation

Adding Analytics to Your Website

Include the tracker in your website's HTML files:

<!-- Add to your website's HTML -->
<script src="/analytics/tracker.js"></script>

The tracker will automatically start collecting data when the page loads.

Tracking Custom Events

Track specific user interactions with custom events:

// Track button clicks automatically
<button data-track="signup-button">
    Sign Up
</button>

// Manual event tracking
AnalyticsTracker.trackEvent('ecommerce', 'purchase', 'premium-plan', 99.99);

Accessing the Dashboard

View your analytics data through the dashboard:

URL: http://your-domain.com/analytics/dashboard.html
Default: No password required (add .htaccess protection for production)

Maintenance & Monitoring

Log Rotation

Set up log rotation to prevent unlimited file growth:

sudo nano /etc/logrotate.d/analytics

Performance Monitoring

Monitor Apache and PHP performance for large-scale deployments.

Backup Strategy

Regularly backup your analytics.log file and configuration.

Security Updates

Keep Apache, PHP, and your system updated with security patches.

Troubleshooting Common Issues

Data not appearing in dashboard

Check browser console for JavaScript errors and verify PHP file permissions.

403 Forbidden errors

Verify .htaccess configuration and file permissions in /analytics/ directory.

High server load

Implement rate limiting in log_endpoint.php and consider log rotation.
